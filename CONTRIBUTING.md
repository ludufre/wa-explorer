# Contributing Guidelines

Thank you for your interest in contributing to this project.  
Contributions are welcome, but please read this document carefully before submitting any pull request or other contribution.

---

## 1. Code of Conduct

By participating in this project, you agree to abide by the rules defined in the
`CODE_OF_CONDUCT.md` file. Respectful, constructive, and professional behavior
is required in all interactions.

## 3. Contribution Scope

Contributions may include:

- Bug fixes
- Performance improvements
- Refactoring
- Documentation improvements
- Tests
- Build or tooling enhancements

Before starting large or structural changes, please open an issue to discuss
the proposal first.

---

## 4. Pull Request Guidelines

- Keep pull requests focused and minimal.
- One logical change per pull request.
- Follow existing code style and conventions.
- Clearly describe **what** was changed and **why**.
- Reference related issues when applicable.

Pull requests that do not follow these guidelines may be rejected or requested
to be revised.
